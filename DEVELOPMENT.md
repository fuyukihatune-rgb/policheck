# 開発ノート（設計判断と進め方）

このプロジェクトをどう作ったか、なぜその設計にしたかを記録する。属人化を避け、
意思決定の背景を残すことが目的。詰まった点の詳細は [README の「8. 詰まった点」](./README.md#8-詰まった点解決プロセス) も参照。

---

## 進め方

- **1機能ずつ、動かしながら**：各ステップで実際に `bun run` して結果を確認し、合意してから次へ。
  一気に全部書いて「動かないけど量だけある」状態を避けた。
- **スコープを固定**：開発指針（`CLAUDE.md`、ローカル保持）にスコープ・出力制約・やらないことを明文化し、
  実装が横に広がるのを防いだ。
- **Claude Code の活用**：論点分割エージェント・RAG・MCP連携を段階的に実装。設計の分岐は都度言語化して記録。

開発順（おおまかに）：RAG（埋め込み・法令取得・検索）→ 出力スキーマ・プロンプト →
論点分割エージェント本体 → サンプルと過検出チューニング → CLI → MCP → HTTP/デプロイ → テスト。

## 主要な設計判断

### RAG / 埋め込み
- **埋め込みは768次元へ縮約＋L2正規化**：数百チャンク規模で3072次元は過剰。正規化済みベクトルなら
  コサイン類似度を単純な内積で計算でき、検索が軽量・単純になる。
- **外部ベクトルDBを使わず自前コサイン**：個情法のみ＝数百チャンクなら全件メモリ総当たりで十分。
  小さく完結させ、依存を増やさない判断。
- **法令は e-Gov 一次情報から取得**：本則185条を整形し、出典URL・取得日・改正版ID（`law_revision_id`）を
  メタに記録（改ざん混入・版ズレの防止）。

### エージェント（論点分割 × Tool Use ループ）
- **単発RAGにしない**：論点ごとに条文検索を回す必要があるため、ループが構造的に必然。
  論点分割 → 各論点で `search_regulation` → LLM照合 → 不十分ならLLM提案クエリで再検索、を自前実装。
- **中核6論点を常時評価**：論点分割をLLM任せにすると実行ごとに揺れ、越境移転などの中核論点を
  取りこぼす回があった。個情法の中核6論点（利用目的／第三者提供／越境移転／開示等請求／窓口／安全管理）は
  必ず評価し、LLMが見つけた固有論点（要配慮・個人関連情報等）を最大4件まで合流させて取りこぼしを防いだ。

### 「断定させない」を構造で強制
- **出力スキーマ検証**：4点セットの形式・必須項目に加え、断定語（適法／違法／罰金◯円）と、
  RAGで未取得の条文（幻覚）を検証で拒否。プロンプト任せにせず、注入で破られても形式外を弾く。
- **過検出の抑制**：中核的義務の欠落のみを挙げ、付随的な記載改善や「やっているかもしれない」前提の
  推測リスクは挙げない方針をプロンプトに課し、low重大度は閾値未満として非表示に。
  結果、`bad=6 / decent=0 / tricky=2`（越境移転が最上位）と明確に差別化。

### LLM プロバイダ
- **埋め込みはGemini、LLM推論はAnthropic Claude**：当初Gemini単独で実装したが、`gemini-2.5-flash`
  無料枠の日次リクエスト上限で、多数のLLM呼び出しを伴う論点分割エージェントが完走できなかった。
  LLM推論のみClaude（`claude-sonnet-4-6`）へ移行し、`LLM_PROVIDER` 環境変数で切替可能な
  プロバイダ抽象化を導入。埋め込みはGeminiのまま（RAGはGemini次元で構築済み）。
- **temperature=0**：コンプライアンス点検は再現性を優先。

### 連携・配布
- **CLI / MCP / HTTP の3エントリで同一エージェントを共有**：まずCLIでE2Eを通し、MCP（公式SDK・stdio）と
  Hono HTTPを薄く被せる構成。MCP Apps の UI（`ui://`）はスコープ外とし、ツール公開に集中。

## 非弁配慮（弁護士法72条）

「AIに断定させない」という設計が、そのまま非弁リスクの回避と一致する。法的判断（鑑定）をせず、
個別事案への当てはめをせず、一般的な観点と関連条文の提示に留め、最終判断は専門家へ橋渡しする。
語も「判定／合法性チェック」を避け「一次点検／論点の洗い出し」で統一している。

## テスト

APIキー不要の純関数（出力スキーマ検証・条番号正規化・インジェクション無害化）を `bun test` で検証。
セキュリティと「断定させない」の中核ロジックを優先的にカバーしている。
